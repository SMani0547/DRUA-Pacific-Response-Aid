# Refinement Plan

## 1. Data model (`src/lib/mock-data.ts`)

Extend `PriorityReport` with optional fields (keeps CSV import working):

- `status`: `"New" | "Under Review" | "Approved" | "Dispatched" | "Resolved"` — defaults to `"New"`.
- `urgency`: `"Immediate" | "Within 6h" | "Within 24h"` (for intake form).
- `roadAccess`: `"Open" | "Limited" | "Blocked"`.
- `description`: free text from intake.
- `reporterType`: `"Community" | "District Officer" | "NGO" | "Hospital"`.
- `previousRiskScore`, `previousSeverity` — snapshot from prior update (for change summary).
- `updatedAt`: ISO string.

Add a `computeRiskScore(input)` helper used by both intake form and CSV import so scoring is one source of truth.

Seed existing 7 mock reports with realistic `previousRiskScore` and `status` so the change summary and command board have something to show on load.

### 2a. Incident Intake Dialog
`<Dialog>` triggered from ImportBar. Form fields: area, issue, severity, people affected, urgency, road access, resources, reporter type, description. On submit:
- `computeRiskScore` → risk 0–100.
- Prepend to `reports` state with `status: "New"` and `updatedAt: now`.
- Toast into the existing `importNotice` slot ("New report from <reporter> added — <area> · risk <score>").

### 2b. AI Response Action Plan
Rendered inside `AreaDetailsSheet` under the existing recommended action:
- Numbered plan (3–5 steps) derived deterministically from the report (severity, people, road access, resources) — labeled "Generated by Gemini AI Agent" to match the existing narrative. No new server call; keeps this a UI/mock refinement.
- Each step: icon + short imperative + ETA badge.

### 2c. Human Approval Workflow
Two surfaces:
- **In the sheet:** status pill + row of buttons `Approve`, `Send to Review`, `Mark Dispatched`, `Mark Resolved`. Clicking updates the report's `status` and `updatedAt` in Dashboard state; sheet header pill re-colors.
- **In the priority table:** new "Status" column with a compact `<Select>` per row (same state).

### 2d. Situation Change Summary
Card above the priority table listing every report where `riskScore !== previousRiskScore` or `severity !== previousSeverity` or `status` changed this session. Each entry shows area, delta (`+7`, `-4`, arrow up/down, red/green), and a one-line rationale ("People affected updated to 3,600" / "Approved for dispatch"). Includes an updated-time stamp.

### 2e. Command Board
A responsive grid grouped by status columns (New · Under Review · Approved · Dispatched · Resolved). Each card shows area, severity pill, risk score, urgency, and a "View" button that opens the sheet. This is a Kanban-style visual complement to the table — no drag-and-drop.

## Technical notes

- All new UI reuses existing shadcn primitives (`Dialog`, `Select`, `Card`, `Badge`, `Button`). No new deps.
- Approval status lives in Dashboard state; no persistence yet (matches current prototype scope).
- Action plan and change rationale are generated client-side deterministically to keep the prototype offline-capable; the "Generate Gemini Summary" button remains the real network call.
- Optional `PriorityReport` fields keep CSV import and the existing MCP tools working without changes.
